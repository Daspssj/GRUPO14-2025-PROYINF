### ✨ Guía para completar la plantilla:  
1. **Descripción**:  
   - Usar formato "Como/Quiero/Para" centrado en valor de usuario.  
   - Ejemplo:  
     ```  
     **Como** docente de matemáticas  
     **Quiero** filtrar preguntas por nivel de dificultad  
     **Para** crear ensayos diferenciados por curso  
     ```  

2. **Criterios de Aceptación**:  
   - Listar funcionalidades concretas (implícitamente Gherkin).  
   - Evitar términos técnicos; usar lenguaje de usuario.  

3. **Valor**:  
   - Vincular siempre a métricas de negocio o pedagógicas.  

4. **Notas Técnicas**:  
   - Incluir solo lo relevante para el equipo de desarrollo.
